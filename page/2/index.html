如何解决Pr2020出现的系统兼容性问题

打开Pr2020之后，出现下面的提示：

  1. 点击“修复”，跳到下面的页面

  2. 点击“NVIDIA驱动程序下载”页面，跳到下面的页面

  3. 根据自己的电脑情况选择1~5中的内容，其中下载类型包括两种，分别是Studio驱动程序(SD)和Game Ready驱动程序(GRD)，前者适用于视频剪辑，后者适用于游戏。

  4. 下载安装完成之后，重启Adobe软件即可解决问题

git远程操作

git 支持本地协议，我们可以在本地初始化一个本地的远程服务器

git init ~/git-server –bare将当前仓库初始化为一个裸仓库

裸仓库没有工作目录，中央服务器不需要工作目录

git remote add aliasName repository：之后我们就可以通过这个aliasName别名来操作远程仓库，远程仓库一个叫做origin。

git remote -v：查看远程仓库的信息

cat .git/config：可以查看所有的配置名称

提交之前先pull然后再提交，否则会出现![reject]这种提示

将信息提交到远程其实是提交历史的复制

git push origin master：将本地的内容推送到远程（origin）的master分支，origin/master这个指针发生了移动，同时，远程仓库的HEAD和master也移动了。

git pull= git fetch + git merge

每一个远程分支在本地都有一个指针，比如远程的master分支在本地的名字叫做origin/master，他不会随着提交移动，只是远程分支在本地的指针。如果checkout到这个引用下面，然后commit并没有什么用，origin/master并不会移动。

git merge origin/master：这个是说将origin/master分支合并到master分支上面
