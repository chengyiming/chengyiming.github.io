git使用

1. 查看所有常见的git命令

$ git

2. 子命令帮助文档

git help

git -h

git –help

man git-

配置git

git config

创建仓库之前就要完成

git config –global user.name “”

git config –global user.email xxx.com

这些用户配置会显示再提交记录中

配置级别介绍：

–local 默认，高优先级： 只会影响本仓库，在.git/config文件中

–global 中等优先级：影响素有当前用户的git仓库 c盘的用户目录下面 /.gitconfig

–system低优先级: 影响所有用户的所有git仓库/etc/gitconfig

初始化仓库：

git init

初始化之后，会形成一个.git文件，它记录了我们所有的提交信息和引用关系

查看仓库的信息

git status

如果没有初始化仓库会提示没有.git这个文件夹

git add

添加文件内容到暂存区，同时文件被跟踪

new file 表示第一次被跟踪，从untrack状态变成了track状态，这是文件级别的变化

git add .

批量添加多个文件

如果出现了不想进行track的文件，并且已经添加到了暂存区中，我们需要进行忽略：

.gitignore的作用：

  1. 在添加时忽略匹配的文件
  2. 仅仅作用于未被追踪的文件

相当于是在进行add的时候进行了一层过滤。

如果文件已经通过add 进行了追踪，该如何删除？

gitignore文件的写法：可以参考github上面的写法：

从暂存区删除文件:

git rm –cached：仅仅从暂存区删除

git rm :从暂存区和工作目录删除

git rm $(git ls-files –deleted)： 删除所有被跟踪，即在暂存区被记录了，但是在工作目录被删除了的文件

git ls-files：到暂存区查找相关的文件

git是一个基于内容寻址的系统，它记录的是内容并不是文件

文件只要在暂存区，那它就是被追踪的状态

git commit

提交：根据暂存区的内容创建一个提交记录

如图所示的三种状态：

  1. 状态1：已经添加到了暂存区，但是还没有被提交
  2. 状态2：已经被track的文件在本地工作空间发生了修改，但是修改还没有提交到暂存区
  3. 状态3：表示还没有被添加到暂存区，没有被track

疑问：提交之后，暂存区还有这些已经提交的文件吗？

git commit -a -m “”

这个命令可以免除使用 git add，直接添加到暂存区，然后再提交（前提是对于已经被追踪了的文件）

git log

只有处于暂存区的内容才能够被提交上去，git是基于内容进行管理的系统，所以说同一个文件可以处于两个不同的区域。

第一行是每一个提交对应的SHA-1编码

git log --oneline这样只会显示比较短的SHA-1编码

配置日志的显示格式：

git log –color –graph –pretty=format:

‘%Cread%h%Creset -%(yellow)%d%Creset %s %Cgreen(%cr)%C(bold blue)<%an>%Creset’ –abbrev-commit

它的显示结果可以表现出分支的合并过程

目前为止，我们通过git config进行了若干设置，这些设置都可以在.git/gitconfig中进行查看。

目前进行了一个提交以及三个设置，查看.git目录：

显示不同版本之间的差异：

git diff ： 工作目录和暂存区之间的差异

git diff --cached [reference]：暂存区和某次提交之间的差异，默认是HEAD，head指向当前的提交

git diff <reference>工作目录和某次提交之间的差异

git diff commit1 commit2:查看两次体加之间的变化

git diff --cached Head:查看head执行的当前提交和暂存区之间的差异，这个顺序是可以反着的吧

git diff:查看本地工作空间和暂存区之间的差异

git diff commit1：查看commit1和工作目录之间的差异

撤销本地的修改：注意这个不是版本的回退

git checkout --file将文件内容从暂存区复制到工作目录，当工作目录有改动，但是还没有提交到暂存区，会丢弃更改，小心使用。

git reset HEAD <file> 取消文件内容的暂存状态，将文件内容从上次提交复制到暂存区

git checkout HEAD --file将内容从上次提交复制到工作目录
